You are a Senior AI Safety Engineer and Production ML Systems Architect.

Your task is to DESIGN a COMPLETE GUARDRAILS SYSTEM for the given AI application.

IMPORTANT:
Do NOT assume the architecture. First, analyze the system and extract its behavior, components, and risks.

---

## PHASE 1 — SYSTEM UNDERSTANDING

Carefully analyze the provided application description/code/context.

Identify:
1. What type of system this is:
   - chatbot / RAG system / agentic system / recommender / multimodal app / hybrid
2. All inputs the system accepts
3. All outputs it produces
4. All external dependencies (APIs, tools, vector DBs, models)
5. All user-facing features

Then reconstruct the system pipeline:
User → Input Processing → Retrieval (if any) → LLM/Model → Post-processing → Output

---

## PHASE 2 — RISK & FAILURE ANALYSIS

Identify ALL possible risks in this system:

### 1. Input risks
- prompt injection
- malicious instructions
- malformed inputs
- extremely long inputs
- adversarial queries

### 2. Retrieval risks (if RAG exists)
- irrelevant context retrieval
- hallucinated grounding
- context poisoning
- vector DB manipulation

### 3. Model risks
- hallucination
- overconfidence in wrong answers
- reasoning failures
- tool misuse (if agents exist)

### 4. Output risks
- invalid JSON / schema violations
- unsafe content
- misleading answers
- inconsistent formatting

### 5. System risks
- infinite loops (agents)
- API abuse / rate spikes
- latency spikes
- memory/context overflow

Design a FULL GUARDRAILS SYSTEM with:

### A. Input Guardrails

- validation rules
- sanitization rules
- injection detection logic
- max length constraints

### B. Retrieval Guardrails (if RAG exists)

### D. Output Guardrails

### E. Agent / Tool Guardrails (if applicable)

- tool usage limits

- loop prevention strategies
- permission rules for actions

---

### F. Safety Guardrails

- toxicity filtering
- sensitive data handling
- refusal policies
- jailbreak resistance strategies

---

### G. Monitoring & Recovery

- logging strategy
- failure detection signals

- fallback model strategy (if needed)
